# run_dialogue.sh 起動時のバタバタ現象 — 原因と対処

## 現象

- `run_dialogue.sh` 起動直後：ロボットがバタバタする
- コマンド送信後：ロボットが宙に浮いたまま動く

## 根本原因（統一的な説明）

**バタバタも浮きも、同じ一つの原因。**

```
start=True を ZMQ 送信
  → WBC が「安全モード」→「SONIC ZMQ 制御モード」に切り替え
  → SONIC が体を「自分のデフォルト立ち高さ H_sonic」に持っていく
  → H_sonic がシミュレーターの床面より高い（高さキャリブレーションのずれ）
  → ロボットが宙に浮いた状態になる
  → 浮いたまま SONIC がバランス制御を続ける → 足が空中で動く → バタバタに見える
```

バタバタは「関節の振動」ではなく「空中でバランスを取ろうとしている動き」。

## 証拠

- `run_deploy.sh` だけの状態（安全モード）：ロボットが床面に立っている → 安定
- `start=True` 送信後（SONIC ZMQ モード）：ロボットが浮く → バタバタ
- mode=0/mode=2 などの planner コマンドは浮きに無関係
  - mode=0 に変更しても、mode=2 に変更しても、バタバタは変わらなかった

## 対処方法

### 運営チームへの報告（根本解決）

SONIC の nominal standing height がシミュレーターの床面レベルと合っていない。
`gear_sonic_deploy` の SONIC 高さキャリブレーション設定を調整してもらう必要がある。

### dialogue.py 側でできること

`send_planner()` の `height` パラメータを `-1.0`（デフォルト使用）から
具体的な値に変更することで調整できる可能性がある。
ただし適切な値は SONIC の設定を見ないと不明。

```python
# 現在（デフォルト使用）
data += struct.pack("<ff", speed, -1.0)

# 変更案（値は運営チームに確認が必要）
data += struct.pack("<ff", speed, <適切な高さ[m]>)
```

## 現在の状態

- 起動時バタバタ：`start_planner()` を遅延初期化に変更して抑制（dialogue.py 修正済み）
  - 最初のコマンドが来るまで `start=True` を送らない → 起動直後は安全モードで安定
  - 最初のコマンド時に1回だけバタバタ → 以降は SONIC モードで継続
- 浮き（高さずれ）：SONIC 設定の問題のため dialogue.py 側では未解決
